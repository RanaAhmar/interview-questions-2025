## 🌟 The 2025 Meta

Interviews have shifted away from solving obscure algorithms on a whiteboard. Instead, companies want to know:
- How do you design scalable APIs?
- Do you understand the tradeoffs of SSR vs CSR?
- Can you integrate AI and LLMs safely into an existing product?

## 📂 Question Bank

### 1. 🧠 AI & LLM Integration (New for 2025)
*   **"How do you prevent prompt injection in a user-facing chatbot?"**
*   "Explain the difference between RAG (Retrieval-Augmented Generation) and fine-tuning an LLM. When would you use each?"
*   "How do you handle rate-limiting and streaming responses from standard LLM APIs?"

### 2. 🏗️ System Design
*   "Design an analytics dashboard that handles 100k events per second."
*   "When does microservices architecture become a liability rather than an asset? Give practical examples."
*   "Explain distributed tracing and how you would implement it across a Next.js frontend and Python backend."

### 3. 🌐 Frontend (React / Vue / Next.js)
*   "Explain React Server Components and how they differ from Server-Side Rendering (SSR)."
*   "How do you optimize Core Web Vitals (LCP, CLS, INP) in a large single-page application?"
*   "What are the security implications of using `dangerouslySetInnerHTML` or similar frameworks constructs, and how do you mitigate them?"

### 4. ⚙️ Backend & API Design
*   "Compare gRPC, GraphQL, and REST. When would you choose one over the other?"
*   "How do you manage database migrations in a CI/CD pipeline with zero downtime?"
*   "Explain how JWTs work and their security pitfalls if used for session management."

## 📚 Study Guides

Each directory in this repository contains detailed answers, mental models, and real-world scenarios to help you structure your responses using the **STAR** method (Situation, Task, Action, Result).

Fork this repository, grab a coffee, and work through one section per week prior to your interviews.
